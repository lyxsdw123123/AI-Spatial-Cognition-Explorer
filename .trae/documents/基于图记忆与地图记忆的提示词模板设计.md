## 目标
- 明确两种记忆模式下传给AI的提示词结构与内容，严格只使用你指定的字段。

## Graph记忆提示词模板
- 用途：基于拓扑连通性与POI相对关系的推理与路径判断。
- 提示词示例：
```
你是一个只依据图结构记忆做推理的AI。严格使用下述数据回答问题，不使用坐标或任何未提供信息。

[模式]
- type: graph

[数据约束]
- 节点(nodes)：仅包含{id, type}，type ∈ {poi, road_node}
- 边(edges)：仅包含{road_id, length_m, from_id, to_id}
- POI关系(poi_relations)：仅包含{poi_a_id, poi_b_id, direction_deg, distance_m, road_id}

[图节点汇总]
- 节点数: {nodes_count}
- POI节点数: {poi_nodes_count}
- 道路节点数: {road_nodes_count}

[图边列表]
{# 避免过长，可按需截断或分页 #}
- edges:
{for each edge}
  - road_id: {road_id}, from: {from_id}, to: {to_id}, length: {length_m}m

[POI相对关系]
- relations:
{for each relation}
  - {poi_a_id} → {poi_b_id}: 方向 {direction_deg}°, 距离 ≈{distance_m}m, 道路: {road_id or 无}

[回答规则]
1) 仅使用图的连通性与POI相对关系；不得使用坐标、类型或主观假设。
2) 路径比较时，以edges的length_m累加为距离依据；若relations出现road_id，表示存在直接道路连接。
3) 方向使用relation的direction_deg（0–360°）。
4) 输出只包含结论，不引入未提供字段。

[问题]
- {question_text}
```

## Map记忆提示词模板
- 用途：基于栅格全局结构（i,j）和道路格分布的空间推理与全局关系判断。
- 提示词示例：
```
你是一个只依据地图式记忆做推理的AI。严格使用下述栅格数据回答问题，不使用地理坐标或任何未提供信息。

[模式]
- type: map

[数据约束]
- 节点(nodes)：仅包含{id, name, type, i, j}，type ∈ {poi, road_node}
- 道路(road_grid)：仅包含{grid_size, cells:[{i,j}]}

[栅格参数]
- grid_size: {grid_size}

[节点列表]
- nodes:
{for each node}
  - {id} ({name}) [{type}] @ ({i},{j})

[道路格子]
- road_cells_count: {cells_count}
- road_cells_sample(≤30):
{for sample cells}
  - ({i},{j})

[回答规则]
1) 仅使用节点的(i,j)与道路格cells；不得使用坐标或主观假设。
2) 判断连通与可达性时，以cells覆盖的格为“道路”；非cells为“非道路”。
3) 相对位置以栅格差值：Δi, Δj；距离可用曼哈顿或栅格步长估算（说明所用方法）。
4) 输出只包含结论，不引入未提供字段。

[问题]
- {question_text}
```

## 生成流程
- Graph提示词：
  - 后端取`GET /memory/graph`快照，统计nodes/edges/relations计数，拼入模板填充。
- Map提示词：
  - 后端取`GET /memory/map`快照，将nodes完整列出；`road_grid.cells`做计数并抽样展示，拼入模板填充。

## 注意事项
- 严格字段：不向提示词加入坐标、类型细节、语义属性等未指定信息。
- 长度控制：edges与cells过长时做分页或抽样；不影响AI推理规则说明。
- 与现有评估兼容：可并行保留“普通记忆”提示词；本两模板用于图与地图模式下的独立评估。
