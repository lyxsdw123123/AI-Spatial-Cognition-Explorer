## 目标
- 在现有系统中新增两种记忆模式：Graph memory 与 Map memory，严格按你的字段约束。
- 保持原“普通记忆（探索上下文提示词）”不变；新增接口用于获取两种记忆快照。
- 前端可切换记忆模式并查看快照摘要。

## 数据结构（严格遵照要求）
- Graph memory（拓扑图）：
  - nodes: [{id, type}]，type ∈ {"poi","road_node"}
  - edges: [{road_id, length_m, from_id, to_id}]
  - poi_relations: [{poi_a_id, poi_b_id, direction_deg, distance_m, road_id|null}]
- Map memory（栅格地图）：
  - nodes: [{id, name, type, i, j}]（type ∈ {"poi","road_node"}）
  - road_grid: {grid_size: 30, cells: [{i, j}, ...]}

## 后端实现点位
- ai_agent/path_memory/path_memory_manager.py
  - 新增：build_graph_memory_snapshot()：
    - 从 `poi_layer.poi_connections` 收集 POI 节点；从 `node_layer.node_sequences.*.nodes` 收集道路节点；
    - 从 `road_layer.road_segments` 生成 edges（road_id=segment_id，长度=length，from_id/start_node，to_id/end_node）；
    - 从 `poi_connections` 计算 POI-POI 相对方向与距离；若存在“直接 POI→POI 边”则 road_id，否则为 null；
  - 新增：build_map_memory_snapshot(boundary, grid_size=30)：
    - 将边界最小外接矩形映射为 30×30 栅格坐标；
    - 将已知 POI 与具有有效坐标的道路节点转换为 `{id,name,type,i,j}`；
    - 将 `road_segments.coordinates` 映射到栅格，使用中心占比法 + 线段采样标注 `cells[{i,j}]`。

- ai_agent/explorer_agent.py
  - 新增字段：`memory_mode: str`（"context"|"graph"|"map"），默认 "context"。
  - 新增方法：
    - set_memory_mode(mode: str)
    - get_memory_summary()：返回 `{mode, summary_counts, graph_snapshot或map_snapshot}`，仅包含定义字段；

- backend/main.py
  - 新增接口：
    - POST `/memory/mode` body: `{mode}` → 设置 `ExplorerAgent.set_memory_mode`
    - GET `/memory/graph` → 返回 `PathMemoryManager.build_graph_memory_snapshot()`
    - GET `/memory/map` → 返回 `PathMemoryManager.build_map_memory_snapshot(boundary, 30)`；boundary 取 `ExplorerAgent.exploration_boundary`
  - WebSocket：`broadcast_status` 附带 `{type:"memory_snapshot", mode, data}`（可选）。

## 前端 app.py
- 侧边栏新增“记忆模式”选择（普通/图/地图），调用 `/memory/mode` 切换；
- 图记忆：显示节点计数、边计数、POI关系条数；
- 地图记忆：叠加 30×30 栅格，着色 `road_grid.cells`，在格中心标注 `nodes` 名称（只读展示）。

## 关键算法说明
- 方向与距离：使用现有 Haversine 与方位角计算（与 explorer_agent 的实现一致）。
- 道路格栅判定：
  - 线段采样：将每段坐标映射到栅格并沿线填充对应格；
  - 中心占比法：若线段距离格中心 < 阈值（如一个格对角长度的 0.25），记为道路格。

## 验证
- 单元测试新增：
  - Graph：节点/边/POI关系字段校验与计数；
  - Map：栅格映射正确性（边界映射、节点落格、道路 cells 非空）。
- 手动运行：北京天安门预设区域，开始探索、停止后调用 `/memory/graph` 和 `/memory/map` 查看结构与计数。

## 不变与约束
- 不修改“普通记忆”的上下文提示词与评估流程。
- 严格遵守字段，不向 Graph/Map 结构写入未指定属性（如经纬度、类型细节等）。
